# Tech

## 使用的技术

- **HTML5**: 用于构建页面结构。
  - `<video>`: 用于嵌入和播放视频。
  - `<audio>`: 用于嵌入和播放音频。
  - `<meta name="google" content="notranslate">`: 用于阻止浏览器自动弹出翻译提示。
- **CSS3**: 用于页面样式。
  - `position: fixed`, `transform`, `object-fit`: 用于实现背景视频的布局和缩放。
  - `transition`: 用于实现文本内容的渐隐渐显动画。
- **JavaScript (ES6)**:
  - `DOM` 操作: 用于获取和修改页面元素。
  - `setInterval`: 用于创建定时循环，以固定间隔切换情诗。
  - `Promise` 和 `catch`: 用于优雅地处理音频自动播放失败的情况。

## 开发设置

- **本地服务器**: 使用 `python3 -m http.server` 或类似的工具在本地运行，以供 `chrome-devtools` 进行测试。
- **浏览器**: 一个现代的网页浏览器，用于打开 `index.html` 文件。

## 技术限制与解决方案

- **音频自动播放**:
  - **限制**: 现代浏览器禁止在用户交互前自动播放带声音的媒体。
  - **解决方案**: 脚本会尝试自动播放。如果失败，它会设置一个一次性的 `click` 事件监听器，在用户首次点击页面时播放音频。这是为了确保兼容性和用户体验的最佳实践。
- **浏览器翻译**:
  - **问题**: 当 `<html>` 标签的 `lang` 属性与页面实际内容语言不符时，浏览器可能会弹出翻译提示。
  - **解决方案**: 将 `lang` 属性设置为正确的语言（如 `zh-CN`），并添加 `<meta name="google" content="notranslate">` 标签来彻底禁用翻译功能。

## 依赖项

- **系统字体**: 优先使用系统内置的楷体（如 `Kaiti SC`, `KaiTi`）来保证轻量和速度，避免了外部网络请求。

## 工具使用模式

- **开发**: 直接在文本编辑器中编辑 `index.html`。
- **测试**:
  - 通过在浏览器中直接打开文件进行快速预览。
  - 通过启动本地服务器，并使用 `chrome-devtools` MCP 工具进行自动化端到端测试，以验证音频播放等交互行为。
